Email Encryption (OpenPGP)

This page tells you exactly what you need to do to send signed and encrypted email.

What encryption does (and doesn’t) do
  • Encrypt = only someone with the private key can read the message content.
  • Sign = proves the message came from the holder of the signing key (and wasn’t altered in transit).
  • Not hidden: email metadata (to/from, date, and usually the subject) is still visible to providers.

If you don’t have a key yet

  2. Create a new key pair (one-time setup).
  3. Make a revocation certificate and back up your key.
  4. Then follow the “already have a key” steps.

If that sounds like a lot: it’s mostly one-time setup. After that, encrypting is just a checkbox.

Pick your tool

You need two things: an email client, and OpenPGP support (built-in or via a key manager). These are common choices:

Desktop (Windows / macOS / Linux): Thunderbird
  • Thunderbird can manage OpenPGP keys and send signed/encrypted mail directly.
  • Best for: simplest setup, least moving parts.
  • Download: thunderbird.net
Windows (optional): Gpg4win + any mail client
  • Gpg4win provides GnuPG plus tools like Kleopatra for key management.
  • Best for: you already use Kleopatra, or you want a separate key manager.
  • Download: gpg4win.org
Android: OpenKeychain + a compatible email app
  • OpenKeychain acts as your key manager. Some email apps can use it for OpenPGP.
  • Best for: mobile sending/reading.
  • Project: openkeychain.org

If you’re unsure: use Thunderbird on desktop. It’s the smoothest “start here”.

2

Create your key (and back it up)

Your key pair has a public key (share it) and a private key (protect it). Treat your private key like a house key: anyone who has it can read your encrypted mail and impersonate your signatures.

Recommended settings (practical)
  • Use your software’s default modern key type unless you need legacy compatibility.
  • Set a strong passphrase you can actually remember.
  • Set an expiry (you can extend later).
Do these two safety steps
  1. Create a revocation certificate and store it offline (USB kept safe, printed QR/backup, etc.).
  2. Back up your private key securely (encrypted storage, offline copy).

If you lose your private key and have no backup, you cannot read old encrypted messages.

Thunderbird: where the key creation usually lives
  • Account Settings → End-to-End Encryption (or OpenPGP) → Add/Generate key.
  • After creation, export your public key to share and export your private key to back up.
Command line (GnuPG): basic key generation

This is one common approach. Your distro may recommend a slightly different flow. 

# Generate a new key (guided)
gpg --full-generate-key

# List keys
gpg --list-secret-keys --keyid-format long

# Export your public key (share this)
gpg --armor --export "Your Name" > my-public-key.asc

# Export your private key (BACK UP securely)
gpg --armor --export-secret-keys "Your Name" > my-private-key.asc

Send a signed & encrypted email

Once keys are in place, sending encrypted mail is just a couple of toggles.

What I prefer
  • Sign + Encrypt when possible.
  • If you can’t encrypt yet, at least sign your first message.
  • Attach your public key so I can reply encrypted.
Message tips
  • Use a neutral subject line. (Subjects usually aren’t encrypted.)
  • Prefer plain text over fancy HTML when troubleshooting.
  • Don’t put secrets in the subject or attachment filenames.
Thunderbird: the usual flow
  1. Compose a new email to marc@fromgravelhill.ca
  2. Turn on Encrypt and Digitally Sign (the wording varies by version).
  3. If prompted, choose your OpenPGP key for signing.
  4. Send. If Thunderbird warns it can’t find my key, import it and verify it (Step 3).

Troubleshooting & common gotchas

“I can’t encrypt - it says there’s no recipient key.”
  • Make sure you imported my public key and that your software sees it as valid.
  • Verify the fingerprint, then mark the key as verified/trusted.
  • Some clients match by email address; use marc@fromgravelhill.ca for the first test.
“My signature shows as ‘unknown / not trusted’.”
  • That’s normal until a key is marked as verified.
  • Verify the fingerprint through a second channel, then set the key as trusted/verified.
“It worked once, then broke.”
  • Check whether your key expired; extend it if needed.
  • If you created a new key, tell your contacts - old encrypted mail won’t decrypt with a new key.
  • If you restored from backup, make sure the key IDs match what your client expects.
Privacy note

OpenPGP protects message content, but email still leaks metadata. If your situation demands stronger anonymity or less metadata leakage, consider a different medium for highly sensitive topics.
