Verify Downloads

If you download something important (installers, APKs, ZIPs, ISOs, finance tools, encryption tools), verify it. This page shows the practical “how” for checksums and signatures.

Open checksum tool SSH & GitHub keys
The goal
  • Confirm the file you downloaded is exactly what the publisher released.
  • Detect corruption, tampering, or “wrong file from the wrong place”.

Table of contents

1) Checksums (fast, common)
2) Signatures (stronger when available)

1) Checksums (fast, common)

A checksum is a hash of the file. If the publisher’s checksum matches your checksum, you have the same bytes they published.

Prefer SHA-256
  • SHA-256 is the typical modern checksum.
  • SHA-1/MD5 are legacy and not recommended for strong integrity guarantees.
Windows (PowerShell)

SHA-256:

Get-FileHash -Algorithm SHA256 .\YourFile.zip

Older Windows option:

certutil -hashfile .\YourFile.zip SHA256
Linux 
sha256sum YourFile.zip
macOS 
shasum -a 256 YourFile.zip
Using the in-browser tool
  1. Open /checksum/
  2. Select the file
  3. Keep algorithm on SHA-256
  4. Paste the publisher’s expected hash to compare

The file never uploads - it stays on your device.

2) Signatures (stronger when available)

If a download includes a signature file (often .sig, .asc, or .gpg), that’s better than a plain checksum - because the publisher’s private key is required to create a valid signature.

Important
  • A signature is only meaningful if you have the right public key and you verified its fingerprint.
  • Don’t just grab a random key and assume it’s the real one.
Typical GPG verification flow
  1. Import the publisher’s public key (from their official site).
  2. Verify the key fingerprint (second channel if possible).
  3. Verify the signature:
gpg --import publisher-public-key.asc
gpg --fingerprint "Publisher Name"
gpg --verify YourFile.zip.asc YourFile.zip

If the signature verifies and the key is trusted/verified, you can be confident the file is authentic.

3) Simple rules that prevent most mistakes

  • Only trust checksums/signatures from the publisher’s official page (not a reposted blog).
  • Make sure the checksum matches the exact filename/version you downloaded.
  • If you get a mismatch: delete the file and re-download from the official source.
  • For high-risk software: prefer signed releases.
  • Remember: a checksum on a compromised website can’t protect you.
